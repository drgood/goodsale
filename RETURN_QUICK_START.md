# Return Management System - Quick Start Guide\n\n## For Shop Managers\n\n### Creating a Return (from Sales page)\n\n1. Navigate to **Sales** from sidebar\n2. Find the sale you want to process a return for\n3. Click the **\"...\"** menu on the right\n4. Select **\"Create Return\"**\n5. Modal appears:\n   - ☐ Check the items you want to return\n   - Reason (optional): \"Defective\", \"Wrong size\", etc.\n   - Refund Method: Cash / Store Credit / Card / Mobile Money\n6. Click **\"Create Return\"**\n7. Success message appears\n8. Return appears in **Returns** page with status \"Pending\"\n\n### Approving/Rejecting Returns (Returns page)\n\n1. Navigate to **Returns** from sidebar\n2. Filter by status if needed (\"Pending Approval\")\n3. Find the return you want to process\n4. Click **\"Details\"** to see full info\n5. Click **\"Approve\"** or **\"Reject\"**\n   - Add notes (optional)\n6. Click **\"Confirm\"**\n7. Return status updates to \"Approved\" or \"Rejected\"\n\n### Processing Refunds (Returns page)\n\n1. Navigate to **Returns** from sidebar\n2. Find an \"Approved\" return\n3. Click **\"Process Refund\"**\n4. Select refund method:\n   - **Cash**: Will be deducted from expected drawer cash\n   - **Store Credit**: Added to customer's account balance\n   - **Card**: Reference only (customer's bank processes)\n   - **Mobile Money**: Reference only (mobile provider processes)\n5. Click **\"Confirm\"**\n6. Return status changes to \"Refunded\"\n7. If Store Credit:\n   - Customer balance increases\n   - Can use balance to pay future purchases\n\n### Configuring Return Policy\n\n1. Navigate to **Settings** from sidebar\n2. Click **\"Return Policy\"** card\n3. Configure:\n   - **Return Window**: How many days customers can return\n   - **Refund Method**: Original payment / Store credit / Let customer choose\n   - **Restocking Fee**: Percentage deducted from refunds\n   - **Require Approval**: Must manager approve first?\n   - **Allow Partial Returns**: Can customer return some items from order?\n   - **Notify Customer**: Send notifications to customer?\n4. Click **\"Save Policy\"**\n\n---\n\n## Return Status Flow\n\n```\nPENDING (Created)\n   ↓\n   ├─→ APPROVED (Manager approved)\n   │      ↓\n   │      └─→ REFUNDED (Refund processed)\n   │           └─→ Complete\n   │\n   └─→ REJECTED (Manager rejected)\n        └─→ Complete\n```\n\n---\n\n## Key Metrics Explained\n\n### Return Amount\n- Total value of items being returned\n- Example: 2 items × GH₵50 = GH₵100\n\n### Restocking Fee\n- Deducted from return amount\n- Based on policy percentage\n- Example: GH₵100 × 10% = GH₵10\n\n### Refund Amount\n- What customer actually gets back\n- Formula: Return Amount - Restocking Fee\n- Example: GH₵100 - GH₵10 = GH₵90\n\n---\n\n## Common Scenarios\n\n### Scenario 1: Customer returns defective item for cash\n1. Customer brings item\n2. Manager: Sales → find sale → Create Return\n3. Select item → Add reason \"Defective\" → Refund Method: Cash\n4. Returns page: Approve the return → Process Refund → Cash\n5. Customer receives cash at counter\n6. System tracks it\n\n### Scenario 2: Store credit for future purchase\n1. Customer initiates return\n2. Manager: Sales → Create Return → Refund Method: Store Credit\n3. Returns page: Approve → Process Refund → Store Credit\n4. Customer's account balance increases\n5. Customer can use balance on next purchase\n6. No cash leaves the drawer\n\n### Scenario 3: Multiple items, partial return\n1. Customer bought: Item A + Item B + Item C\n2. Only wants to return Item B\n3. Manager: Create Return → Uncheck A & C → Check B only\n4. Calculate: Only B amount shown\n5. Continue with approval flow\n\n### Scenario 4: Wrong refund method chosen\n1. Return is \"Approved\"\n2. Manager realizes wrong refund method selected\n3. Go back to Returns page\n4. Can still \"Process Refund\" and change method\n\n---\n\n## Tips & Best Practices\n\n✅ **DO:**\n- Document return reason (helps identify patterns)\n- Check policy before promising refund\n- Use Store Credit when possible (keeps cash in business)\n- Review returns daily\n- Adjust policy based on return patterns\n\n❌ **DON'T:**\n- Promise refunds before manager approval\n- Process cash refunds without documentation\n- Forget to update policy when needed\n- Leave returns in \"Pending\" state indefinitely\n\n---\n\n## Troubleshooting\n\n### Return isn't showing up\n- Check Returns page filters (make sure status isn't filtered)\n- Refresh page\n- Check that sale exists in Sales page\n\n### Can't select item\n- Modal must be fully loaded\n- Try refreshing\n- Check sale has valid items\n\n### Refund amount seems wrong\n- Check your store policy restocking fee %\n- Formula: (Sale Amount) × (1 - Restocking Fee %)\n- Example: GH₵100 × (1 - 10%) = GH₵90\n\n### Customer balance not updating\n- Must use \"Store Credit\" refund method\n- Return must be \"Refunded\" status\n- Check Returns page to confirm status changed\n\n---\n\n## Keyboard Shortcuts\n\n| Action | Keys |\n|--------|------|\n| Open Sales | Alt+S |\n| Open Returns | Alt+R |\n| Open Settings | Alt+T |\n| Search | Ctrl+F |\n\n---\n\n## Support\n\n- **Questions?** Check Returns Management page help\n- **Bug reports**: Contact system admin\n- **Policy questions**: Refer to Return Policy configuration\n"
